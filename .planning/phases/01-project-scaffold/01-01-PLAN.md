<task type="auto">
  <name>Task 1: Scaffold Astro 5 project and configure Tailwind v4, TypeScript, and Netlify</name>
  <files>astro.config.mjs, tsconfig.json, netlify.toml, src/styles/global.css, src/pages/index.astro, .gitignore, package.json</files>
  <action>
Run the following scaffold command from the repo root. The `.` target installs into the current directory (repo root). `--no-git` is critical — the repo already exists, do NOT reinitialize git:

```bash
npm create astro@latest . -- --template minimal --typescript strictest --no-git --skip-houston
```

If prompted to overwrite existing files, only allow it to overwrite non-planning files. Verify `.planning/` exists and is intact after the command completes before proceeding.

After scaffold completes, install Tailwind v4 and add the sitemap integration:

```bash
npm install tailwindcss @tailwindcss/vite
npx astro add sitemap
```

When `npx astro add sitemap` runs, accept the auto-patch to `astro.config.mjs`. After it patches, verify the file still contains `@tailwindcss/vite` — the `astro add` command may replace the vite plugins block. If it was removed, restore it.

**Final astro.config.mjs must be exactly:**
```typescript
import { defineConfig } from "astro/config";
import tailwindcss from "@tailwindcss/vite";
import sitemap from "@astrojs/sitemap";

export default defineConfig({
  site: "https://workwiser.netlify.app",
  integrations: [sitemap()],
  vite: {
    plugins: [tailwindcss()],
  },
});
```

Note: `site: "https://workwiser.netlify.app"` is a placeholder URL. Without the `site` property, `@astrojs/sitemap` silently produces no sitemap. The real Netlify subdomain will be confirmed after first deploy — update this value if Netlify assigns a different subdomain.

**tsconfig.json must be:**
```json
{
  "extends": "astro/tsconfigs/strictest"
}
```
Do not add extra compilerOptions — the `strictest` preset already includes everything needed.

**Create netlify.toml in the repo root:**
```toml
[build]
  command   = "npm run build"
  publish   = "dist"

[build.environment]
  NODE_VERSION = "22"
```

NODE_VERSION = "22" is required. Astro 5 requires Node v22.12.0+; Netlify's default is lower and will fail with a cryptic syntax error without this.

**Create src/styles/global.css:**
```css
@import "tailwindcss";
```

This single line is the entire Tailwind v4 entry point. Do NOT add `@tailwind base; @tailwind components; @tailwind utilities;` — that is the Tailwind v3 syntax and will not work with v4.

The global.css does not need to be imported anywhere yet — Phase 2 (BaseLayout) will import it. For now it just needs to exist.

**Update .gitignore** to include:
```
dist/
.output/
node_modules/
.env
.env.*
!.env.example
.DS_Store
```

**Create public/robots.txt:**
```
User-agent: *
Allow: /

Sitemap: https://workwiser.netlify.app/sitemap-index.xml
```

Note: The Sitemap URL uses the same placeholder as astro.config.mjs. Update both after the real Netlify subdomain is confirmed.

After all files are written, run `npm run build` to verify zero errors. If build fails, do NOT proceed to Task 2.

Commit all changes:
```
git add astro.config.mjs tsconfig.json netlify.toml package.json package-lock.json src/pages/index.astro src/styles/global.css public/robots.txt .gitignore
git commit -m "feat(01-01): scaffold Astro 5 with Tailwind v4, sitemap, and Netlify config"
```

Do NOT use `@astrojs/tailwind` — it is deprecated for Tailwind v4 and will cause build errors.
Do NOT run `npx astro add tailwind` — it may install the deprecated v3-style integration.

Option B (manual):
1. Create a GitHub account at https://github.com if you do not have one
2. Create a new repository named `workwiser-website`
3. Run from the repo root:

5. Confirm build command is `npm run build` and publish directory is `dist` (matches netlify.toml)
