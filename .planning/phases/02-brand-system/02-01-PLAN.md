<task type="auto">
  <name>Task 1: Define brand tokens and configure font loading</name>
  <files>src/styles/global.css, astro.config.mjs</files>
  <action>
1. Update `src/styles/global.css` to add the `@theme {}` block AFTER the existing `@import "tailwindcss"` line. Define these tokens:
   - `--color-navy: #003c64;`
   - `--color-blue: #0072c9;`
   - `--color-teal: #26aeb4;`
   - `--color-teal-light: #effefe;`
   - `--color-yellow: #f3f145;`
   - `--font-heading: 'Open Sauce One', sans-serif;`
   - `--font-body: 'Inter', sans-serif;`

   Also add base body styles below @theme:
   ```css
   @layer base {
     body {
       @apply font-body text-navy;
     }
     h1, h2, h3, h4, h5, h6 {
       @apply font-heading;
     }
   }
   ```

2. Update `astro.config.mjs`:
   - Change the import to: `import { defineConfig, fontProviders } from 'astro/config';`
   - Add `fonts` array to the config with two entries:
     - Open Sauce One via `fontProviders.fontsource()`, cssVariable `'--font-open-sauce-one'`, weights `[700, 800]`, styles `['normal']`
     - Inter via `fontProviders.fontsource()`, cssVariable `'--font-inter'`, weights `[400, 500]`, styles `['normal']`

   NOTE: The `@theme` font-family values ('Open Sauce One', 'Inter') must match what the Astro fonts API loads. The cssVariable names in astro.config are separate from the @theme variables — Astro uses them for `<Font />` preload tags; Tailwind uses the @theme --font-* variables for utility classes.

3. After making changes, run `npm run build` to verify. If the Astro fonts API fails to resolve "Open Sauce One", fall back to direct npm install:
   ```bash
   npm install @fontsource/open-sauce-one @fontsource-variable/inter
   ```
   Then add imports ABOVE `@import "tailwindcss"` in global.css:
   ```css
   @import '@fontsource/open-sauce-one/700.css';
   @import '@fontsource/open-sauce-one/800.css';
   @import '@fontsource-variable/inter';
   ```
   And remove the `fonts` array from astro.config.mjs. Do NOT use both methods simultaneously.
  </action>
  <verify>
    <automated>npm run build</automated>
  </verify>
