The BaseLayout already has `title: string` and `description?: string` props destructured from `Astro.props`, with a default description of "WorkWiser — Your Trusted Virtual Assistant Partners". Use `Astro.url` for the canonical URL.

Add these exact meta tags (per user decision in 08-CONTEXT.md):

```html
<!-- Open Graph -->
<meta property="og:title" content={`${title} | WorkWiser`} />
<meta property="og:description" content={description} />
<meta property="og:image" content={new URL('/images/hero.jpg', Astro.site ?? Astro.url).href} />
<meta property="og:url" content={Astro.url.href} />
<meta property="og:type" content="website" />

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content={`${title} | WorkWiser`} />
<meta name="twitter:description" content={description} />
<meta name="twitter:image" content={new URL('/images/hero.jpg', Astro.site ?? Astro.url).href} />
```

Also add a canonical link tag for SEO best practice:
```html
<link rel="canonical" href={Astro.url.href} />
```

IMPORTANT: The `site` property must be set in astro.config.mjs for `Astro.site` to work. Check if it exists; if not, add `site: 'https://workwiser.io'` to the Astro config object. If it already has a site value, leave it as-is.

Do NOT change the Props interface, do NOT change the title format, do NOT change existing meta tags. Only ADD the new OG/Twitter tags and canonical link.
